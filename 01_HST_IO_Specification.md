# 01_HST_IO_Specification — I/O 仕様の定義と本リポジトリの想定 I/O

**正本（Canonical）**: 日本語版。

本ドキュメントは、**I/O Specification（入出力仕様）** の定義を明確にし、本リポジトリ（holographic-sphere-topology）が想定する In/Out について注記する。

---

## 1. I/O Specification の定義

### 1.1 定義

**I/O Specification（入出力仕様）** とは、ある Object（系・モジュール・ブラックボックス）に対して、**どのような Input（In）に対して、どのような Output（Out）を出力するか** を記述した仕様である。

| 要素 | 説明 |
|------|------|
| **Input (In)** | 系に投入される問い・データ・トリガー。境界面の上流側 |
| **Output (Out)** | 系が返す回答・結果・応答。境界面の下流側 |
| **I/O Specification** | In と Out の対応関係を定義する文書・図・型・契約 |

### 1.2 関連用語

| 用語 | 文脈 | 説明 |
|------|------|------|
| **Interface Specification** | システム・API | 境界での入出力の取り決め。I/O Specification と同義的に用いられる |
| **Contract** | 設計・プログラミング | Design by Contract における事前条件（In）・事後条件（Out） |
| **Type Signature** | プログラミング | 関数の引数型と戻り値型（例: `A → B`） |
| **Protocol** | 通信・API | やり取りの形式と順序の定義 |

### 1.3 HST における位置づけ

HST において、I/O Specification は **境界面（boundary）上の型・契約・プロトコル** に相当する。内部を開かずに、境界面上の In/Out の対応関係を定義することで、目的適合性を認証できる。

---

## 2. 本リポジトリが想定する I/O

本リポジトリ（holographic-sphere-topology）は、[00_HST_Basic_Concepts.md](00_HST_Basic_Concepts.md) で定義された通り、**事業という複雑系の外部・内部境界面をどう整理するか** についてのクエリに対する回答を出力する Object である。

### 2.1 想定される Input (In)

| 種類 | 内容 |
|------|------|
| **クエリ** | 事業という複雑系の**外部・内部境界面をどう整理するか**に関する問い |
| **例** | 「システムと DB の境界はどこに置くべきか」「財務と営業の境界面はどう設計するか」「モノリシックを臓器分離するには」「系内部と系外部の責務をどう分けるか」「資本 Input → 純利益 Output の球にどう丸め込むか」 |

### 2.2 想定される Output (Out)

| 種類 | 内容 |
|------|------|
| **回答** | 上記クエリに対する、HST の規則に則った整理方針 |
| **含まれる要素** | 臓器メタファー、Object/Morphism 設計、境界面の型・契約・プロトコル、系内部/系外部の責務分離（毒の原則）、Zero Knowledge Proof 型認証、GAAS スタックとの対応、影響範囲（既存インフラ・人材・キャリア） |

### 2.3 I/O 対応図

```
[In] 事業という複雑系の外部・内部境界面をどう整理するかについてのクエリ
     │
     ▼
┌─────────────────────────────────────────────────────────┐
│  holographic-sphere-topology (本リポジトリ)              │
│  HST の規則に則った Object                                │
│  00_HST_Basic_Concepts.md 等を参照                        │
└─────────────────────────────────────────────────────────┘
     │
     ▼
[Out] HST に基づく境界面整理の回答
     （臓器メタファー、Object/Morphism、型・契約・プロトコル、
      毒の原則、ZKP 型認証、GAAS スタック対応 等）
```
